### Change 2 — Guard the finally block (PLAY-07 / D-01)

At the `finally` block (currently lines 673-675), change:
- FROM: `_isReinitializing = false;`

This combines the gen-guard (D-01) with the finally cancel-site (D-05.3) in one block. Only the active generation clears the flag AND cancels its settle subscription.

### Change 3 — Cancel `_initSettleSub` at top of next initSongs (PLAY-08 / D-05.1)

Immediately after `final myGen = ++_initGen;` (line 424), add:
`_initSettleSub?.cancel();`

This prevents stacking: the previous init's listener dies before the new one starts.

### Change 4 — Cancel `_initSettleSub` in stop() (PLAY-08 / D-05.2)

In the `stop()` method (lines 936-942), add `_initSettleSub?.cancel();` alongside the existing `_positionUpdateTimer?.cancel();` and `_coverSub?.cancel();`.

### Change 5 — Replace fire-and-forget listener with tracked field (PLAY-08 / D-03, D-04, D-06)

At the listener block (currently lines 603-642):
- Change `final sub = _player.processingStateStream.listen((state) {` to `_initSettleSub?.cancel(); _initSettleSub = _player.processingStateStream.listen((state) {`
- Keep ALL callback logic unchanged (ready re-trigger lines 606-609, idle recovery lines 613-620, 30s buffering-skip lines 624-637 — D-06: preserve exact behavior)
- DELETE the line `Future.delayed(const Duration(seconds: 60), () => sub.cancel());` (D-04)

### Change 6 — Remove orphan listener (PLAY-09 / D-07)

DELETE the orphan listener at line 645:
`_player.processingStateStream.listen((state) { AppLogger.debug('initSongs: player processingState=$state'); });`

This listener logs only and is never cancelled. Line 604 (`AppLogger.debug('initSongs: processingState=$state')` inside the tracked listener) already logs every state change, so the orphan is redundant.

### Change 7 — Keep [DIAG] logs (D-08)
