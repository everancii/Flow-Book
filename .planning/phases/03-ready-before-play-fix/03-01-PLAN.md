# Plan 03-01: Ready-Before-Play Fix

<objective>
THE actual bug fix. Restructure `MyAudioHandler.initSongs` to await `ProcessingState.ready` before calling `_player.play()`, with a bounded 10s timeout. Fix the big play button call site. Surface errors as user-visible SnackBars. After this phase, Sound-Books books auto-play on open — matching all other sources.
</objective>

<context>
All implementation decisions are LOCKED in `03-CONTEXT.md` (D-01 through D-12). The executor must implement exactly these decisions.

**Root cause (confirmed Phase 1):** `_player.play()` fires unconditionally at line 587. When the player is in `ProcessingState.loading`/`buffering` (Sound-Books duration probe in-flight), the fork's `play()` method calls `audioSession.setActive(true)` which fails or reverts — play request is silently dropped.

**The fix:** Gate `play()` behind `await processingStateStream.firstWhere(ready).timeout(10s)`. BehaviorSubject replays last value → synchronous short-circuit for already-ready sources (zero latency). For loading sources (Sound-Books), waits until ready or 10s timeout.
</context>

Replace the entire block currently at lines 584-646 (from `if (playImmediately) {` through the closing `}` before `_waitForStartToSettle`). The new block:

```
if (playImmediately) {
  // D-01: Await ProcessingState.ready before play(). BehaviorSubject
  // replays the last value, so already-ready sources (LibriVox, YouTube,
  // knigavuhe, 4read) complete synchronously — zero added latency.
  // Sound-Books (loading during duration probe) waits until ready or 10s.
  try {
    await _player.processingStateStream
        .firstWhere((s) => s == ProcessingState.ready)
        .timeout(_readyTimeout);
  } on TimeoutException {
    AppLogger.error(
        'initSongs: timed out waiting for ProcessingState.ready after ${_readyTimeout.inSeconds}s');
    rethrow;
  }

  // D-02: Stale init check — a newer initSongs may have started during the await.
  if (myGen != _initGen) return;

  AppLogger.debug(
      'initSongs: ready confirmed, calling play(), state=${_player.processingState}');
  _player.play();
}
```
